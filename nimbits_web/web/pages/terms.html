Using the Public Server

You can download a copy of Nimbits Server and deploy to a Google App Engine account without limitations.

We host an instance of Nimbits Server for all to use on cloud.nimbits.com with some limitations. Most users will not notice these limitations, nor will you if your data recording frequencies are about one a minute. If you intend to use Nimbits to record multiple values a second - particularly values that go out of their compression setting on each change, then you should set up a server of your own for that purpose.

The Nimbits Public server may receive code changes updates that change or limit the way you can access data you've uploaded to the server. We may, without warning limit download / upload quotas based on server load.

How we handle possible over-use of the public server

  1. When we see a very large spike in CPU / Bandwidth usage, we first become very happy to see that someone out there is using our product.
  2. Next, we analyse the traffic to see if the problem is with the server itself, potentially improving the server for everyones benefit.
  3. Then, we do our best to improve whatever resource is being used at such a high level, potentially optimizing the process so the hits no longer show up on our radar.
  4. We will then attempt to contact the owner of the data points being written to using the email address on that account, attempting to understand their needs and how we can help.
  5. We also will offer to assist the owner of the points in setting up their own app engine server instance. We offer free advice and hourly consulting services. We also can support your server with an ongoing service agreement.
  6. If we are unable to contact the owner of the traffic we will, with a heavy heart, blacklist the source IP address.
