The code follows a very robust and expandable architecture

  • nimbits-model contains all of the base model objects and interfaces such as Value, Point, Category etc. The models are Serializable Data transfer objects.
  • nimbits-sdk compiles as a runnable JAR and inherits nimbits-model. It contains many helper functions that access the REST APIs hosted on a nimbits-tds. One might use the SDK to call GetCurrentValue() in the SDK to get a Point's current Value; objects defined in nimbits-model. If one was to just call the /service/currentvalue REST service directly on nimbits server you would get the JSON data of a Value Object. The SDK simply helps authenticate and de-serialize the object for you.
  • nimbits-tds
    • is the nimbits server which contains the com.nimbits.client namespace which is the UI and Async interfaces to the web services and com.nimbits.server which houses the ORM, DAO, Servlets, REST APU and so on.
