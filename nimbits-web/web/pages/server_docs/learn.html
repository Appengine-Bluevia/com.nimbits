OK! Let's get started. Here is Step 1 for getting connected to the cloud.

Nimbits is a big system, with a lot of moving parts. There is a lot of power inside its cloud. We recomend getting started the same way you'd eat an elephant: One bite at a time.

Test drive the public cloud

Here is the link to the administrative console for Nimbits server: http://app2.nimbits.com

It's worth mentioning that we target Google Chrome as the supported browser for Nimbits, but it also works on most standards based browsers such as FireFox. If you're using Internet Explorer, you may be prompted to install Google Chrome Frame which is an add-on that will make Nimbits run faster in IE.

The admin console lets you manage your account, upload and download data manually and create "things". When you log in, you'll be prompted to log in with a Google Account, since that's what we use for this instance's security. This can be a gmail account, but it can also be any email, even your company's email i.e boss@bigcompany.com. Follow the steps for registering your email address with Google or sign up for a google account, and you'll be good to go.

Create a Data Point

A Point represents a thing. You're child's weight, the core temperature of a nuclear power plant, the temperature of an aquarium, nitrogen content on a sensor on your Mars rover. Whatever you have to record. A point is a bucket for recording timestamped Value Objects which contain gps, numbers, xml, or json data.

Everything in Nimbits can be automated using code and our REST API

Entities and Organization

When you log into Nimbits you'll see your account name, which is the top of a tree where you can organise your stuff. When you add a data point, it will appear under your account name.

As you add more stuff, you can drag and drop them around your tree to organize them how you like. You can have many child points under a parent one, or folders, files and so on.

Basic Point Settings

Right click on your new Point and select Edit. Here you can set your point's settings.

  • Compression Filters: Filters give you a way to tell Nimbits to reject data that should not be recorded. Noisy, repetitive data can be screened out.
  • Expire: Tell Nimbits to automatically delete data that is older than you want to keep
  • Unit of Measure: Whatever label you want to apply, such a Watts for an electrical reading.
  • Who can view: determines if anyone can read your data point and find it in the search engine, people who connected account with, or only you.
  • Descriptions: It's important to set a description if you want your point to be easier to find in the search engine.
Alerts

Alerts tell Nimbits that the state of a points will go into a High or Low Alert State when a value comes in above or below a setting. Also, a point can go into a Idle alert state if a point does not receive a new value for a set amount of time.

You can subscribe to points belonging to other users, getting notified of their alerts.
Subscriptions and Notifications

You can subscribe to a point and ask Nimbits to Notify you when events occur. You can configure many subscriptions, getting alerted in different ways based on the settings. Right click on your new Data Point and select Subscribe.

When you create a subscription, it will appear as a new icon under your data point in the tree. You can organize your subscriptions any way you like. Their location in the tree has nothing to do with their settings or what point you're subscribing to. I like to keep all of my subscriptions in a folder. In the image below, you can see how I have two subscriptions to my Aquariums temperature probe, one to alert me if the temp goes too high, another to let me know if a value hasn't been recorded in a while, so maybe my probe is off.

A point can be in one of four states

  • Normal All is well.
  • High: The last recorded number value was higher or equal to the high alert setting.
  • Low: The last recorded number value was lower or equal to the high alert setting.
  • Idle: The point has not received a new value in a set number of time, based on the idle setting.

When a point goes into an Alert State, it will attempt to notify you in ways set in the point's properties.
The following alert methods are available

  • Email
  • Instant Message XMPP
  • Facebook
  • Twitter
  • Your Data Feed Point
  • When using the portal or android app, point icons change color based on the alert state.
You can choose to have the content of an alert sent in JSON format, so other programs can pick it up.

Now that you have a point created, you can being recording data. Think of a point as a bucket filled with timestamped Value Objects. You can store much more than a series of numbers in a point. Each recorded value can contain:

  • A Number: e.g. a temperature, or any decimal value.
  • A Timestamp: Stored in UTC time
  • GPS Coords
  • Text String: Notes, used in annotations etc.
  • Data: Any string data, such as XML or JSON
