Value Object

Think of a data point in nimbits as a bucket, and we fill it with Value Objects. As Values go into the bucket, alerts, stats and other events are triggered. Further, there are rules on what can go into the bucket based on the Point's settings like filters that say only values within a certain range can go in.

A Value Object can consists of the following:

  • Timestamp: saved in UTC format on the cloud
  • Double Value: such as 1.23
  • GPS Coordinates
  • Note: A string note
  • Data: String data, such as xml or JSON
  • State: the alert state, such as high, low or idle.

To save space in storage and on the wire, we abbreviate the names of the above fields.

  • ts: Timestamp
  • d: The double value
  • lt, ln: Latitude and Longitude
  • n: Note
  • dx: Data
  • st: State

When Sending and receiving raw Value Object, they will come to you in JSON format.

A single value

{"lt":0.0,"lg":0.0,"d":3.14,"t":1342373434789,"n":"some notes","dx":"some data","st":3}
