This guide is meant to help get you started when you are running your own Instance of Nimbits on Google App Engine™.

Cost of Ownership

Running an instance of Nimbits on Google App Engine requires an account with Google where you can set a quota for each day allowing Google to bill you directly for resources consumed.

When you consume resources on Google's infrastructure, Google charges will apply. Please see Google App Engine Billing and Budgeting.

We do everything we can to reduce the cost of processing data in our code. One of the most expensive things you can do in Google App Engine is to repeatedly access the data store. This is why we heavily cache data in memory, buffer incoming data, use advanced techniques such as data sharding and try to only touch the data store using batch operations. Should you notice an operation you are using on the server causing a large consumption of quotas and resources, please contact support asap. There is an excellent chance that we can identify the cause and significantly reduce the cost of the process you are using.

Out of the box, Nimbits is configured to run fast vs cheap. Some steps you can take to reduce the cost of your instance include the following. Also, if you are using an instance of Nimbits under a support agreement, just contact support to request these options to be set for you.

  • Disabling Sessions - Sessions data is used rarely. You can test your instance with sessions disabled by changing sessions-enabled in appengine-web.xml to false. You can then also remove the session maintenance cron job from cron.xml
  • Idle settings - Nimbits polls points with Idle Alarms set once every 5 minutes. If you don't require that level of monitoring, the setting can be changes in cron.xml
  • App Engine settings - log into the Admin Console and select Application Settings. There, you can manage how many new instances Nimbits will create, the type of memory and CPU those instances have, and other options to reduce performance but also reduce cost. Optionally, you can use these settings to increase Nimbits' performance with an increase to the cost of ownership.
