Debian / Ubuntu Users

Edit the /etc/apt/sources.list file to add the Nimbits repository to the bottom: 

deb http://deb.nimbits.net/packages/ ./

at a terminal, run the following commands: 

sudo apt-get update
sudo apt-get install nimbits

Edit the file /opt/nimbits/server/war/WEB-INF/appengine-web.xml and change the app id to your Nimbits Instance. Note the version Number you will need to set to the current version on the App Engine Console. Then run the following command from the bin directory in your App Engine SDK: 

        ./appcfg.sh update /opt/nimbits/server/war

Depending on your needs, you can use our hosted nimbits server right now on http://app.nimbits.com, or deploy your own to Google App Engine. You can also build an entire nimbits ecosystem with clusters of virtualized servers inside your companies network that can also then relay data out to your choice of externally hosted clouds. Nimbits Server can run on a laptop, server cluster, Google App Engine or Amazon's EC2 cloud.

Nimbits Servers can be interacted with using a collection of REST Web Services where you can do things like record and download data. There is also an SDK for working with your Nimbits Environment, the SDK simplifies using the REST services and handles things like authentication for you. It is a Java program you can include in your development projects or run independently from the command line.

Nimbits server is built on Cloud Computing architecture which makes it highly scalable. You can run Nimbits on a single virtual machine and continue to add virtual machines as your resource needs increase. Nimbits will scale across any number of machines, sharing memory, CPU and I/O resources seamlessly.
