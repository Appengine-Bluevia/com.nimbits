Nimbits integrates into the WolframAlpha computational Knowledge Engine (BETA)

WolframAlpha is a computational knowledge engine with vast data stores of information and the ability to interpret complex queries and mathematical formulas.

BETA

Please note that this feature is currently in BETA and subject to change and my become temporarily unavailable.

Nimbits integrates into WolframAlpha, allowing you to configure a data points to trigger a computation whenever it receives a new value. The result is then stored in a target Point. The result is that you can add intelligence to your software and hardware projects, using Nimbits' ability to relay, share and store your data streams and WolframAlpha's ability to understand and compute it.

Example 1: Basic Math

I have two data points called "foo" and "bar" and i'm feeding a temperature in fahrenheit into it. Whenever foo gets a new value, I want to run an equation to convert foo's value to celsius and store the result in "bar"

Logging into Nimbits, i can right click my data point named "foo" and select the "Intelligence" tab. Here, i enter the formula for converting fahrenheit to celsius, but for the value i put the point name.value in brackets: ([foo.value] -32) * (5/9)

Now whenever foo gets a new value recorded, the value will get converted and stored in bar. You can try your formula first on wolframalpha.com, or find more advanced formulas on http://www.wolframalpha.com/examples/ For example phi([foo.value]) will get the results of this number-theoretic functions

Example 2: String Data

A Nimbits Data Point stores a history of Value Objects. Value objects can contain a number value, like the celsius value above. A GPS coordinate, an Annotation (text), and a raw data value (JSON Or XML) that can be any sort of data you like. In fact the Nimbits SDK allows you to directly store Programming Object directly into a Point.

In This example, whenever foo is written to, we're going to store the result as string data in the point, instead of a number. This example shows getting the current stock price for Google and storing it as data in another point.

Note the option selected to record the result as text data and the Pod Id to pull the Quote section from the response we get from WolframAlpha. Pod Id tells Nimbits what portion of the response you want to store in your point. You can learn more about it here http://products.wolframalpha.com/api/documentation.html

Finally, checking off that I want the result in plain text gives me: 

[$561.37(GOOG | NASDAQ | Friday 12:59:58 pm EST | 2 days 7 hrs ago)]

Unchecking the "Result in plain text box" gives me the data in XML.
