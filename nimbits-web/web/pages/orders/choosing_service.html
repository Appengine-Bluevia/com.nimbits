Choosing a deployment

Nimbits is the only platform for the Internet of Things that allows you to use a public cloud that we host, your own instance on a cloud provider like App Engine or Amazon EC2, or on your own infrastructure. We also provide support for all of these options. This guide can help you decide which type of service is right for you.

Easiest! The Public Cloud

We host this instance and manage the monitoring and upgrades. We handle the cost of ownership and provide a free quota for users only requiring a small amount of API calls a day. Uploading and downloading data is capped by a free quota, with the option of adding API credits to your account. You don't need to do anything to get start, just Login.

Private Cloud

You create an instance on AppSpot.com and download and install nimbits yourself or order one of our support packages.

Reasons to do this:
  • You can control when and how your Nimbits instance is upgraded and patched.
  • You can have more granular control over your usage with access to the App Engine admin console
  • Privacy, your completely own your data without anyone in between, even us.
Internal Instance using AppScale

You can run a Nimbits Instance on your own infrastructure using a product called AppScale. The setup is somewhat complex, requiring KVM based virtual machines running linux. Nimbits can then be scaled across many servers internally.

Reasons to do this:
  • Complete ownership of the hardware, security and data
  • Using your existing infrastructure.

A private instance using AppScale is a complicated setup - please contact us to discuss support options.
